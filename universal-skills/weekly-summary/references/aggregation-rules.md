# 周报聚合规则

## 数据来源优先级

1. **已保存的日报 JSON**: `~/.cursor/oh-my-skills/daily-summaries/{date}.json`
2. **回退**: 如果某些天没有日报，直接从 Cursor 原始数据获取（使用 fetch_sessions.py + fetch_conversations.py）

## 聚合维度

### 整体统计
- 总会话数 = 各天 total_sessions 之和
- 有效工作天数 = 有日报数据的天数
- 代码变更总量 = 各天变更之和

### 工作类型分布
- 按 categories 字段聚合，各类型计数求和

### 关键成就提取
- 从每天的日报 summary 中提取核心工作成果
- 合并同一项目的相关工作
- 突出完成的里程碑和重要功能

### 项目维度
- 按项目名称分组
- 列出每个项目的主要工作和变更

## 周报风格

- **面向上报**: 简洁、突出成果、量化进展
- **项目导向**: 按项目组织而非按日期
- **要点式**: 使用 bullet points
- **可量化**: 包含代码行数、会话数、文件数等数据
