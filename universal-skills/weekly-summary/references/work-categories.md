# 工作类型识别规则

根据对话内容识别工作类型，每个会话归类为一个主要类型。

## requirements_discussion (需求讨论)

**关键词**: 需求、讨论、方案、设计、规划、确认、proposal、design
**场景**: 讨论功能需求、设计方案、技术选型、架构决策

## coding (编码)

**关键词**: 实现、编写、开发、添加、创建、修改、implement、write
**场景**: 实现新功能、编写代码、修改文件、添加组件

## problem_solving (问题解决)

**关键词**: bug、error、issue、fix、排查、调试、debug、exception
**场景**: 修复 bug、排查问题、调试错误、解决异常

## refactoring (重构)

**关键词**: 重构、优化、改进、清理、整理、refactor、optimize
**场景**: 代码重构、性能优化、代码清理、架构调整

## code_review (代码审查)

**关键词**: review、检查、评审、建议
**场景**: 代码审查、PR Review、质量检查

## documentation (文档)

**关键词**: 文档、注释、README、文档、说明、document
**场景**: 编写文档、添加注释、创建 README

## testing (测试)

**关键词**: test、测试、单元测试、集成测试、验证
**场景**: 编写测试、运行测试、测试覆盖率

## other (其他)

无法明确归入以上类型的工作。
