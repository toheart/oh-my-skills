# 用户画像分析维度

## 编码风格 (Coding Style)

从用户消息中分析:
- **命名习惯**: camelCase / snake_case / PascalCase 偏好
- **架构偏好**: 微服务/单体、DDD/MVC、函数式/面向对象
- **注释风格**: 中文/英文、详细/简洁
- **代码组织**: 文件粒度、模块化程度
- **错误处理**: 偏好的错误处理模式

## 技术画像 (Technical Profile)

分为三个熟练度层级:
- **Expert**: 深度使用、能提出架构级建议的技术
- **Proficient**: 日常使用、能独立完成任务的技术
- **Learning**: 正在学习、需要更多指导的技术

识别线索:
- 使用的编程语言和框架
- 提到的工具和服务
- 问题的深度和方向

## 沟通风格 (Communication Style)

- **提问模式**: 直接/渐进、具体/概括
- **反馈方式**: 快速确认/详细审查
- **语言偏好**: 中文/英文、技术术语使用程度
- **指令风格**: 命令式/协作式

## 工作习惯 (Work Habits)

- **活跃时段**: 上午/下午/晚上/深夜
- **会话风格**: 深度长会话 vs 快速短会话
- **问题解决**: 自顶向下/自底向上
- **迭代方式**: 增量式/一次性

## 输出格式

```markdown
# 用户画像

## 编码风格
- [具体习惯和偏好]

## 技术画像
- **Expert**: [语言/框架列表]
- **Proficient**: [语言/框架列表]
- **Learning**: [语言/框架列表]

## 沟通风格
- [提问和反馈模式]

## 工作习惯
- 活跃时段: [时段]
- 会话风格: [深度/快速]
- 偏好: [其他偏好]
```
