# Testing Specification

This specification defines unit testing and integration testing standards for the project, following TDD (Test-Driven Development) principles.

Go 测试使用 **Ginkgo v2 + Gomega** 作为测试框架，使用 **mockery** 生成 mock 文件。

## TDD Development Process

### Red-Green-Refactor Cycle

1. **Red**: Write failing tests first, define expected behavior
2. **Green**: Write minimal code to make tests pass
3. **Refactor**: Refactor code while keeping tests passing

### Test-First Principle

- Write test cases before developing new features
- Write tests to reproduce bugs before fixing them
- Ensure existing test coverage when refactoring

### File Organization

- Test files in the same directory as source files, named `*_test.go`
- Each public function/method should have corresponding tests
- Mock files generated by mockery to `mocks/` subdirectory

```
domain/example/
├── entity.go
├── repository.go
├── mocks/                    # mockery 生成的 mock 文件
│   └── mock_repository.go
├── entity_test.go
└── service_test.go
```

### Ginkgo Test Structure

每个测试文件需要一个 `TestXxx` 入口函数注册 Ginkgo：

```go
package example

import (
	"testing"

	. "github.com/onsi/ginkgo/v2"
	. "github.com/onsi/gomega"
)

func TestExample(t *testing.T) {
	RegisterFailHandler(Fail)
	RunSpecs(t, "Example Suite")
}
```

### Describe / Context / It 组织测试

使用 `Describe` 描述被测对象，`Context` 区分场景，`It` 定义具体断言：

```go
var _ = Describe("ExampleService", func() {
	var (
		svc  *ExampleService
		repo *MockRepository
	)

	BeforeEach(func() {
		repo = NewMockRepository()
		svc = NewExampleService(repo)
	})

	Describe("GetByID", func() {
		Context("ID 存在时", func() {
			BeforeEach(func() {
				repo.On("GetByID", mock.Anything, "test-1").Return(&Example{ID: "test-1", Name: "Test"}, nil)
			})

			It("should return the example", func() {
				result, err := svc.GetByID(context.Background(), "test-1")
				Expect(err).NotTo(HaveOccurred())
				Expect(result).NotTo(BeNil())
				Expect(result.Name).To(Equal("Test"))
			})
		})

		Context("ID 不存在时", func() {
			BeforeEach(func() {
				repo.On("GetByID", mock.Anything, "not-exist").Return(nil, ErrNotFound)
			})

			It("should return ErrNotFound", func() {
				_, err := svc.GetByID(context.Background(), "not-exist")
				Expect(err).To(MatchError(ErrNotFound))
			})
		})
	})

	Describe("Create", func() {
		It("should create and return the example", func() {
			repo.On("Create", mock.Anything, mock.AnythingOfType("*example.Example")).Return(nil)

			result, err := svc.Create(context.Background(), &CreateRequest{Name: "New"})
			Expect(err).NotTo(HaveOccurred())
			Expect(result).NotTo(BeNil())
			Expect(result.Name).To(Equal("New"))
		})
	})
})
```

### Gomega 常用断言

```go
// 相等
Expect(result).To(Equal(expected))
Expect(result).NotTo(BeNil())

// 错误
Expect(err).NotTo(HaveOccurred())
Expect(err).To(MatchError(ErrNotFound))
Expect(err).To(HaveOccurred())

// 集合
Expect(items).To(HaveLen(3))
Expect(items).To(ContainElement("foo"))
Expect(toolNames).To(ContainElements("query", "list"))
Expect(items).To(BeEmpty())

// 布尔
Expect(result.Active).To(BeTrue())
Expect(result.Deleted).To(BeFalse())

// 数值
Expect(count).To(BeNumerically(">", 0))
Expect(count).To(BeNumerically("<=", 100))

// Panic
Expect(func() { s.Stop() }).NotTo(Panic())

// 字符串
Expect(msg).To(ContainSubstring("error"))
```

## Mockery — Mock 生成

### 配置文件 `.mockery.yaml`

放在项目根目录（`backend/` 下）：

```yaml
with-expecter: true
dir: "{{.InterfaceDir}}/mocks"
outpkg: "mocks"
filename: "mock_{{.InterfaceName | snakecase}}.go"
mockname: "Mock{{.InterfaceName}}"

packages:
  github.com/user/project-name/internal/domain/example:
    interfaces:
      Repository:
```

配置说明：
- `with-expecter: true`：生成类型安全的 expecter API
- `dir`：mock 文件输出到接口所在包的 `mocks/` 子目录
- `mockname`：mock 类型名为 `Mock` + 接口名

### 生成 Mock

```bash
# 在 backend/ 目录下执行
make mock

# 或直接调用
mockery
```

### Makefile 配置

```makefile
# 生成 mock 文件（需要安装 mockery: go install github.com/vektra/mockery/v2@latest）
mock:
	mockery
```

### 使用生成的 Mock

```go
import (
	"github.com/user/project-name/internal/domain/example/mocks"
)

var _ = Describe("ExampleService", func() {
	var (
		svc      *ExampleService
		mockRepo *mocks.MockRepository
	)

	BeforeEach(func() {
		mockRepo = mocks.NewMockRepository(GinkgoT())
		svc = NewExampleService(mockRepo)
	})

	It("should list examples", func() {
		// Expecter API（推荐，类型安全）
		mockRepo.EXPECT().List(mock.Anything).Return([]*Example{{ID: "1"}}, nil)

		results, err := svc.List(context.Background())
		Expect(err).NotTo(HaveOccurred())
		Expect(results).To(HaveLen(1))
	})
})
```

### 手写 Mock（简单场景）

对于简单的 mock 或测试内部使用的辅助接口，可以直接手写：

```go
type mockCacheInvalidator struct {
	invalidated bool
}

func (m *mockCacheInvalidator) InvalidateCache() {
	m.invalidated = true
}
```

## Layered Testing Strategy

### Domain Layer - Unit Tests

- Pure logic tests, no external dependencies
- Mock all repository interfaces using mockery
- High coverage requirement (>80%)

```go
func TestDomainExample(t *testing.T) {
	RegisterFailHandler(Fail)
	RunSpecs(t, "Domain Example Suite")
}

var _ = Describe("ExampleService", func() {
	var (
		svc      *ExampleService
		mockRepo *mocks.MockRepository
	)

	BeforeEach(func() {
		mockRepo = mocks.NewMockRepository(GinkgoT())
		svc = NewExampleService(mockRepo)
	})

	Describe("List", func() {
		It("should return all examples", func() {
			mockRepo.EXPECT().List(mock.Anything).Return([]*Example{}, nil)
			result, err := svc.List(context.Background())
			Expect(err).NotTo(HaveOccurred())
			Expect(result).NotTo(BeNil())
		})
	})
})
```

### Application Layer - Integration Tests

- Test use case orchestration logic
- Mock infrastructure layer
- Verify DTO conversion

### Infrastructure Layer - Integration Tests

- Use real dependencies (temporary database, files)
- Clean up resources after tests
- Can be skipped (CI environment)

```go
var _ = Describe("ExampleRepository [integration]", func() {
	BeforeEach(func() {
		if testing.Short() {
			Skip("skipping integration test")
		}
	})

	It("should save and retrieve", func() {
		// Use real database
	})
})
```

### Interfaces Layer - E2E Tests

- Test HTTP endpoints
- Use `httptest` package
- Verify request/response format

```go
var _ = Describe("ExampleHandler", func() {
	var router *gin.Engine

	BeforeEach(func() {
		router = setupTestRouter()
	})

	Describe("GET /api/v1/examples", func() {
		It("should return example list", func() {
			req := httptest.NewRequest("GET", "/api/v1/examples", nil)
			w := httptest.NewRecorder()
			router.ServeHTTP(w, req)

			Expect(w.Code).To(Equal(http.StatusOK))

			var resp Response
			json.Unmarshal(w.Body.Bytes(), &resp)
			Expect(resp.Code).To(Equal(0))
		})
	})
})
```

## Running Tests

### Go Test Commands

```bash
# Run all tests
make test

# Run all tests with ginkgo CLI (recommended, parallel support)
ginkgo ./...

# Run specific package tests
ginkgo ./internal/domain/example/

# Run with verbose output
ginkgo -v ./...

# Run with coverage
ginkgo --cover ./...
make test-coverage

# Skip integration tests
go test -short ./...

# Generate mock files
make mock
```


## Testing Best Practices

### Do

- Test names describe expected behavior (Chinese is acceptable in Ginkgo descriptions)
- One `It` verifies one behavior
- Keep tests independent, no execution order dependency
- Use `BeforeEach` for shared setup
- Use mockery `EXPECT()` for type-safe mock setup
- Pass `GinkgoT()` to mockery constructors for automatic assertion

### Don't

- Don't test private functions (test through public interfaces)
- Don't use `time.Sleep` in tests (use channels or mocks)
- Don't ignore error return values
- Don't share test state across `It` blocks
- Don't test third-party library behavior
